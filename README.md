# Мультимодальный анализ рентгенограмм грудной клетки

Задача проекта - проверить, помогает ли краткая информация о пациенте улучшить анализ рентгеновского снимка. Модель получает фронтальный снимок грудной клетки и поле `indication` с причиной назначения исследования.

Результат работы модели:

- вероятность класса `abnormal` и решение `normal / abnormal`;
- вероятности пяти наиболее частых патологий;
- отдельные оценки веток изображения и текста.

`normal` означает, что в разметке исследования нет патологических терминов. `abnormal` означает, что в поле `Problems` указано хотя бы одно отклонение. Это исследовательская модель, а не средство постановки медицинского диагноза.

Streamlit используется только как интерфейс для ручной проверки. Основной результат проекта - обученные классификаторы, схема объединения двух источников данных и сравнение метрик.

## Датасет

Используется [Indiana University Chest X-ray Dataset](https://www.kaggle.com/datasets/raddar/chest-xrays-indiana-university):

- `indiana_reports.csv` содержит `uid`, `Problems`, `indication`, `findings` и `impression`;
- `indiana_projections.csv` связывает имя файла с исследованием и проекцией;
- `images/images_normalized/` содержит PNG-снимки.

В проект берутся только снимки с проекцией `Frontal`. Таблицы объединяются по `uid`, после чего по полю `Problems` создаются бинарная метка `is_abnormal` и метки пяти частых патологий:

- Opacity;
- Cardiomegaly;
- Pulmonary Atelectasis;
- Calcinosis;
- Calcified Granuloma.

Поля `findings` и `impression` не подаются модели. Они написаны после интерпретации снимка и часто уже содержат ответ, поэтому их использование дало бы утечку целевой переменной. Из текста используется только `indication`, то есть информация, которая известна до исследования.

После подготовки остается 3818 фронтальных снимков:

| Часть выборки | Количество |
|---------------|-----------:|
| train         |       2672 |
| validation    |        384 |
| test          |        762 |

Разделение выполняется по `uid`. Снимки одного исследования не могут попасть одновременно в обучение и проверку. На `validation` выбираются параметры и порог решения, а `test` используется один раз для итогового сравнения.

## Как работает модель

Снимок переводится в оттенки серого, нормализуется и приводится к размеру `224 x 224`. Предобученная `TorchXRayVision DenseNet121` с весами `densenet121-res224-all` формирует вектор из 1024 признаков.

Из `indication` удаляются маски `XXXX` и лишние символы. Затем `Bio_ClinicalBERT` переводит текст в вектор из 768 признаков. При усреднении учитываются только настоящие токены, без padding.

В финальной версии DenseNet121 и ClinicalBERT не обучаются с нуля
и не настраиваются отдельно для каждого класса, на данном датасете такое переобучение явно ухудшает метрики. Они используются как готовые экстракторы признаков. На полученных векторах обучаются регуляризованные классификационные головы:

- image-only голова по признакам снимка;
- text-only голова по признакам `indication`;
- отдельные бинарные головы для пяти патологий.

Для решения `normal / abnormal` используется image-first каскад:

```text
p_final = sigmoid(1.0 * logit(p_image) + 0.5 * logit(p_text))
```

Это не обычное голосование.
Сначала каждая ветка рассчитывает свою вероятность,
затем вероятности переводятся в `log-odds`
и складываются с выбранными на validation коэффициентами.
Снимок остается главным источником, а текст вносит поправку.
Веса фиксированы для всех случаев.
Выбранные параметры:

| Параметр                 | Значение |
|--------------------------|---------:|
| image regularization `C` |   0.0003 |
| text regularization `C`  |    0.001 |
| image log-odds weight    |      1.0 |
| text log-odds weight     |      0.5 |
| cascade threshold        |    0.265 |
| image-only threshold     |     0.25 |

Если пользователь не вводит текст, применяется image-only голова со своим порогом. Для top-5 патологий векторы DenseNet121 и ClinicalBERT объединяются, после чего каждая патологическая голова рассчитывает собственную вероятность.

## Что сравнивалось

Сначала были проверены простые модели: TF-IDF для текста, уменьшенные пиксели и статистики яркости для снимка, а также раннее и gated-объединение. Затем те же варианты были построены на признаках ClinicalBERT и TorchXRayVision.

Итоговое сравнение проводилось на одних и тех же 762 объектах test split:

| Вариант                     |   ROC-AUC |    PR-AUC |        F1 |
|-----------------------------|----------:|----------:|----------:|
| классический image-only     |     0.682 |     0.768 |     0.677 |
| классический gated fusion   |     0.686 |     0.782 |     0.706 |
| neural early fusion         |     0.727 |     0.813 |     0.727 |
| neural gated fusion         |     0.698 |     0.790 |     0.722 |
| внешний TorchXRayVision     |     0.742 |     0.823 |     0.768 |
| обученная image-only голова |     0.800 |     0.864 |     0.800 |
| итоговый image-first каскад | **0.802** | **0.866** | **0.810** |

Простое объединение больших векторов оказалось слабее image-only модели: поле `indication` короткое, часто повторяется и содержит мало диагностической информации. Поэтому в финальном варианте текст не заменяет снимок, а только корректирует его ответ.

По сравнению с внешним TorchXRayVision итоговый каскад дал:

- `+0.059` ROC-AUC;
- `+0.042` PR-AUC;
- `+0.042` F1.

Важно, что основная часть прироста получена не от текста, а от обучения регуляризованной image-only головы под разметку IU X-ray. Добавление `indication` поверх этой головы дало еще `+0.002` ROC-AUC, `+0.002` PR-AUC и около `+0.010` F1. Прирост небольшой, но он получен без `findings` и `impression`, то есть без утечки ответа.

Для top-5 патологий нейросетевые признаки также оказались лучше классических:

| Метрика       | Классические признаки | Нейросетевые признаки |
|---------------|----------------------:|----------------------:|
| micro ROC-AUC |                 0.629 |                 0.689 |
| macro ROC-AUC |                 0.619 |                 0.678 |
| micro PR-AUC  |                 0.152 |                 0.223 |
| macro F1      |                 0.188 |                 0.269 |

Основная метрика проекта - PR-AUC, так как классы распределены неравномерно и важна работа с положительным классом `abnormal`. ROC-AUC показывает качество ранжирования на всех порогах, а F1 оценивает решение при пороге, заранее выбранном на validation.

модель превзошла выбранный внешний baseline на том же датасете, с тем же test split и той же целевой переменной.

## Связанные работы

Статья М.И. Беляева и соавторов [«Автоматизация анализа рентгеновских снимков грудной клетки с использованием методов глубокого обучения и объяснимого искусственного интеллекта»](https://sanse.ru/index.php/sanse/article/view/662). В ней рассматриваются DenseNet, ResNet, EfficientNet, Grad-CAM и LIME для анализа рентгенограмм.

Наиболее близкая по постановке работа - [Indication as Prior Knowledge for Multimodal Disease Classification in Chest Radiographs with Transformers](https://arxiv.org/abs/2202.06076). В ней клиническое показание также используется вместе со снимком. Код авторов открыт в репозитории [jacenkow/mmbt](https://github.com/jacenkow/mmbt), но их абсолютные метрики нельзя сравнивать с этим проектом из-за другого датасета и набора классов.

Дополнительно  [TieNet](https://arxiv.org/abs/1801.04334) и работа [Multimodal Deep Learning for Integrating Chest Radiographs and Clinical Parameters](https://pubs.rsna.org/doi/10.1148/radiol.230806). Готовая рентгеновская модель и внешний baseline взяты из открытой библиотеки [TorchXRayVision](https://github.com/mlmed/torchxrayvision).

## Запуск

Установка зависимостей:

```powershell
python -m pip install -r requirements.txt
$env:PYTHONPATH="src"
```

Подготовка общей таблицы из исходных CSV и снимков:

```powershell
python -m ml_pr.data.prepare_iu_xray --top-k 5
```

Обучение классических моделей для сравнения:

```powershell
python -m ml_pr.training.baselines --task binary --modalities text image fusion
```

Расчет внешнего TorchXRayVision baseline:

```powershell
python -m ml_pr.training.external_torchxrayvision_baseline --device cuda
```

Обучение финальных голов и подбор каскада:

```powershell
python -m ml_pr.training.train_neural_final `
  --metadata data/processed/iu_xray_metadata.csv `
  --output-dir outputs/neural_final `
  --cache-dir outputs/neural `
  --device auto
```


Запуск интерфейса:

```powershell
streamlit run src/ml_pr/service/app.py
```

После запуска Streamlit открывается по адресу `http://localhost:8501`. В PyCharm папку `src` можно отметить как Sources Root или установить проект командой `python -m pip install -e .`.

## Основные файлы

- `src/ml_pr/data/` - сборка metadata, меток и split;
- `src/ml_pr/features/` - очистка текста и простые признаки для baseline;
- `src/ml_pr/models/neural_sklearn.py` - классификационные головы и каскад;
- `src/ml_pr/training/train_neural_final.py` - обучение финальной модели;
- `src/ml_pr/training/neural_comparison.py` - извлечение признаков ClinicalBERT и DenseNet121;
- `src/ml_pr/inference/neural_predictor.py` - предсказание для одного случая;
- `src/ml_pr/service/app.py` - интерфейс Streamlit;
- `outputs/neural_final/` - модели, метки и итоговые метрики.

Финальные артефакты:
